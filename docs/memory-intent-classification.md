# 记忆意图分类规则

## 概述

记忆意图分类器用于判断用户输入是否需要注入记忆上下文。这是基于规则的轻量级分类，不调用 LLM。

## 分类结果

| 意图 | 说明 | 是否注入记忆 |
|------|------|-------------|
| `none` | 无需记忆 | 否 |
| `short_term` | 需要短期记忆 | 注入会话摘要 |
| `long_term` | 需要长期记忆 | 检索相关记忆 |
| `both` | 两者都需要 | 全部注入 |

## 规则定义

### 无需记忆 (none)

匹配以下模式的用户输入不需要记忆上下文：

```python
NO_MEMORY_PATTERNS = [
    r'^(你好|hi|hello|hey)[\s!！.。]*$',      # 问候语
    r'^(谢谢|thanks|thank you)[\s!！.。]*$',   # 感谢语
    r'^(好的|ok|okay|嗯|嗯嗯)[\s!！.。]*$',    # 确认语
    r'^(再见|bye|goodbye)[\s!！.。]*$',        # 告别语
    r'^(什么是|解释一下|什么是)',               # 知识问答
    r'^(如何|怎么|怎样)',                       # 方法询问
    r'^(写一个|生成|创建)',                     # 内容生成
]
```

**示例：**
- "你好" → `none`
- "什么是 Python？" → `none`
- "写一个快速排序" → `none`

### 需要短期记忆 (short_term)

匹配以下模式需要引用当前会话：

```python
SHORT_TERM_PATTERNS = [
    r'(刚才|刚刚|之前说的|上面提到|之前聊)',    # 引用刚才内容
    r'(继续|接着|下一步|接下来)',               # 任务延续
    r'(我们讨论|我们说|我们聊)',                # 引用对话
    r'(上一个问题|刚才的问题)',                 # 引用问题
    r'(回到|回到刚才)',                        # 返回话题
]
```

**示例：**
- "我们刚才讨论了什么？" → `short_term`
- "继续上一步" → `short_term`
- "你刚才说的那个方法" → `short_term`

### 需要长期记忆 (long_term)

匹配以下模式需要引用历史信息：

```python
LONG_TERM_PATTERNS = [
    r'(上次|以前|之前|曾经|往期)',              # 历史引用
    r'(我的偏好|我喜欢|我不喜欢|我讨厌)',        # 偏好表达
    r'(记住|别忘了|记得)',                      # 记忆请求
    r'(那个项目|那个问题)',                     # 模糊引用
    r'(历史|过去的)',                          # 历史查询
    r'(我的信息|我的资料)',                     # 个人信息
]
```

**示例：**
- "上次我们聊的项目" → `long_term`
- "我喜欢简洁的回答" → `long_term`
- "记住我的邮箱是 xxx" → `long_term`

## 工作流程

```
用户输入
    │
    ▼
┌─────────────────────────────┐
│    匹配 NO_MEMORY_PATTERNS   │
└─────────────────────────────┘
    │ 是              │ 否
    ▼                 ▼
返回 none    ┌─────────────────────────────┐
             │ 匹配 SHORT_TERM_PATTERNS     │
             │ 匹配 LONG_TERM_PATTERNS      │
             └─────────────────────────────┘
                      │
           ┌──────────┼──────────┐
           ▼          ▼          ▼
     仅短期匹配   仅长期匹配   两者都匹配
           │          │          │
           ▼          ▼          ▼
      short_term  long_term    both
```

## 优先级

1. `NO_MEMORY_PATTERNS` 优先级最高
2. 短期和长期模式可以同时匹配，返回 `both`
3. 如果所有模式都不匹配，默认返回 `none`

## 性能考虑

- 分类器使用正则表达式，执行时间 < 1ms
- 不涉及数据库查询或 LLM 调用
- 可以在每轮对话前执行，不影响响应延迟

## 扩展

要添加新的分类规则，编辑 `backend/app/services/memory_context_service.py` 中的 `MemoryIntentClassifier` 类。

添加新模式的建议：
1. 确保模式足够具体，避免误匹配
2. 使用原始字符串 `r'...'` 避免转义问题
3. 测试边界情况（空输入、特殊字符等）
