## 新增需求

### 需求:系统必须使用 `.env` 作为配置的单一真相来源

所有配置项必须定义在项目根目录的 `.env` 文件中，各组件必须从该文件读取配置。

#### 场景:配置文件存在
- **当** 项目根目录存在 `.env` 文件
- **那么** 所有组件必须能够读取其中的配置项

#### 场景:配置文件不存在
- **当** 项目根目录不存在 `.env` 文件
- **那么** 系统必须显示友好的错误提示，建议复制 `.env.example`

### 需求:前端必须从环境变量读取后端地址

`vite.config.ts` 必须通过 `loadEnv` 或 `define` 从环境变量读取 `VITE_API_BASE_URL`，禁止硬编码端口。

#### 场景:从环境变量读取后端地址
- **当** `.env` 文件中定义了 `VITE_API_BASE_URL=http://localhost:8001`
- **那么** 前端代理必须将 `/api` 请求转发到 `http://localhost:8001`

#### 场景:环境变量未定义时使用默认值
- **当** `.env` 文件中未定义 `VITE_API_BASE_URL`
- **那么** 前端代理必须使用默认值 `http://localhost:8001`

### 需求:启动脚本必须从 `.env` 读取配置

`start-backend.ps1` 和 `start-frontend.ps1` 必须解析 `.env` 文件并设置环境变量，禁止硬编码任何配置值。

#### 场景:启动脚本读取端口配置
- **当** `.env` 文件中定义了 `BACKEND_PORT=8001`
- **那么** `start-backend.ps1` 必须在端口 8001 启动后端服务

#### 场景:启动脚本读取 API 地址
- **当** `.env` 文件中定义了 `VITE_API_BASE_URL=http://localhost:8001`
- **那么** `start-frontend.ps1` 必须设置正确的环境变量供 Vite 使用

### 需求:必须提供完整的配置模板

`.env.example` 必须包含所有必需的配置项及其说明，新用户可以通过复制该文件快速配置项目。

#### 场景:配置模板完整性
- **当** 用户查看 `.env.example` 文件
- **那么** 文件必须包含所有必需配置项及其注释说明

#### 场景:从模板创建配置
- **当** 用户复制 `.env.example` 为 `.env`
- **那么** 项目必须能够使用默认配置正常启动

### 需求:必须提供统一启动入口

项目必须提供 `start.ps1` 脚本，一键启动所有服务（前端、后端），无需分别运行多个脚本。

#### 场景:一键启动所有服务
- **当** 用户运行 `start.ps1` 脚本
- **那么** 系统必须依次启动后端服务和前端服务

#### 场景:启动前检查依赖
- **当** 用户运行 `start.ps1` 脚本
- **那么** 系统必须检查 `.env` 文件存在和 PostgreSQL 服务运行状态

#### 场景:启动后显示服务地址
- **当** 所有服务启动成功
- **那么** 系统必须显示前端地址、后端地址、API 文档地址

### 需求:必须提供统一停止入口

项目必须提供 `stop.ps1` 脚本，一键停止所有服务。

#### 场景:一键停止所有服务
- **当** 用户运行 `stop.ps1` 脚本
- **那么** 系统必须停止所有正在运行的后端和前端进程

#### 场景:停止确认
- **当** 所有服务停止完成
- **那么** 系统必须显示已停止的服务列表
