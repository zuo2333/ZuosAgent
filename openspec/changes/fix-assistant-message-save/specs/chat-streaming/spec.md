## 修改需求

### 需求:流式聊天消息持久化

系统必须在流式聊天响应期间正确保存所有消息到数据库，包括用户消息和大模型回复消息。

#### 场景:用户发送消息后，用户消息和回复消息都被保存

- **当** 用户发送一条消息并收到流式回复
- **那么** 数据库中必须同时保存用户消息（role=user）和大模型回复消息（role=assistant）

#### 场景:历史会话加载时显示完整对话

- **当** 用户打开一个历史会话
- **那么** 系统必须加载该会话的所有消息，包括用户消息和大模型回复消息

#### 场景:流式响应中断时，用户消息仍然保留

- **当** 流式响应过程中发生错误或中断
- **那么** 已保存的用户消息必须保留在数据库中
