## 新增需求

### 需求:系统必须支持模型调用工具

系统必须实现 Function Calling 机制，允许模型在对话中调用预定义的工具。

#### 场景:模型决定调用工具
- **当** 模型分析用户请求后决定需要使用工具
- **那么** 系统返回工具调用请求，包含工具名称和参数

#### 场景:工具调用流程
- **当** 模型请求调用工具
- **那么** 系统执行工具并将结果返回给模型继续生成

### 需求:系统必须提供网页搜索工具

系统必须提供 web_search 工具，允许模型搜索互联网信息。

#### 场景:执行网页搜索
- **当** 模型调用 web_search 工具并传入搜索关键词
- **那么** 系统使用 DuckDuckGo 执行搜索并返回结果列表

#### 场景:搜索结果格式
- **当** 网页搜索完成
- **那么** 返回包含标题、链接、摘要的结果列表（默认最多 5 条）

#### 场景:搜索失败处理
- **当** 网页搜索因网络问题失败
- **那么** 系统返回错误信息，模型可尝试其他方式回答

### 需求:系统必须提供数据库查询工具

系统必须提供 db_query 工具，允许模型查询应用自身数据库。

#### 场景:执行数据库查询
- **当** 模型调用 db_query 工具并传入 SQL 语句
- **那么** 系统执行只读查询并返回结果

#### 场景:禁止写入操作
- **当** 模型尝试执行 INSERT、UPDATE、DELETE 等写入语句
- **那么** 系统拒绝执行并返回错误

#### 场景:查询表白名单
- **当** 模型查询未在白名单中的表
- **那么** 系统拒绝执行并返回权限错误

#### 场景:结果行数限制
- **当** 查询结果超过最大行数限制（默认 100 行）
- **那么** 系统截断结果并提示已限制

#### 场景:禁止访问系统表
- **当** 模型尝试查询 pg_ 前缀的系统表
- **那么** 系统拒绝执行并返回权限错误

### 需求:数据库查询表白名单必须可配置

管理员必须能够配置允许查询的表列表。

#### 场景:默认允许的表
- **当** 系统初始化
- **那么** 默认允许查询 sessions、messages、tool_calls 表

#### 场景:查看表白名单
- **当** 用户查看数据库查询工具配置
- **那么** 系统显示当前允许查询的表列表

### 需求:工具执行必须有超时限制

每个工具调用必须在规定时间内完成，超时后强制终止。

#### 场景:网页搜索超时
- **当** 网页搜索执行超过 30 秒
- **那么** 系统终止请求并返回超时错误

#### 场景:数据库查询超时
- **当** 数据库查询执行超过 10 秒
- **那么** 系统终止查询并返回超时错误

#### 场景:记录超时状态
- **当** 工具执行超时
- **那么** 系统记录状态为 timeout，并记录已执行时间

### 需求:系统必须实现会话级工具白名单

每个会话必须能够独立配置允许使用的工具集。

#### 场景:默认工具配置
- **当** 创建新会话
- **那么** 会话默认启用所有可用工具

#### 场景:禁用特定工具
- **当** 用户在会话设置中禁用某工具
- **那么** 该会话中模型无法调用该工具

#### 场景:工具调用权限检查
- **当** 模型请求调用工具
- **那么** 系统检查该工具是否在当前会话的白名单中

#### 场景:权限拒绝处理
- **当** 工具不在白名单中
- **那么** 系统返回权限拒绝错误，模型使用其他方式回答

### 需求:系统必须记录工具调用历史

所有工具调用必须被记录，便于审计和调试。

#### 场景:记录成功调用
- **当** 工具调用成功执行
- **那么** 系统记录工具名称、输入参数、输出结果、状态为 success

#### 场景:记录失败调用
- **当** 工具调用失败或被拒绝
- **那么** 系统记录工具名称、输入参数、错误信息、状态为 failed 或 denied

### 需求:前端必须显示工具调用过程

用户必须能够在对话中看到模型调用了哪些工具及结果。

#### 场景:显示工具调用卡片
- **当** 模型调用工具
- **那么** 前端显示工具调用卡片，包含工具名称和参数

#### 场景:显示工具执行状态
- **当** 工具正在执行
- **那么** 卡片显示加载状态

#### 场景:显示工具结果摘要
- **当** 工具执行完成
- **那么** 卡片显示执行结果摘要（可展开查看详情）

#### 场景:显示工具执行失败
- **当** 工具执行失败或被拒绝
- **那么** 卡片显示错误信息

### 需求:用户必须能够在会话设置中管理工具

前端必须提供界面让用户配置会话的工具权限。

#### 场景:查看可用工具列表
- **当** 用户打开会话设置
- **那么** 显示所有可用工具及其描述

#### 场景:切换工具启用状态
- **当** 用户点击工具开关
- **那么** 该工具在当前会话中的启用状态切换

#### 场景:保存工具配置
- **当** 用户修改工具配置后保存
- **那么** 配置持久化到数据库
