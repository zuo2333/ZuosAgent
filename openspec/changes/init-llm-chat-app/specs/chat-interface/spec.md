## 新增需求

### 需求:用户可以发送消息并接收回复

系统必须提供聊天输入框，允许用户输入文本消息并发送给选定的模型。

#### 场景:发送文本消息
- **当** 用户在输入框中输入文本并点击发送按钮
- **那么** 系统将消息添加到对话历史，并调用模型生成回复

#### 场景:发送快捷键支持
- **当** 用户按下 Enter 键（未按住 Shift）
- **那么** 系统发送当前输入框中的消息

#### 场景:多行输入支持
- **当** 用户按下 Shift+Enter
- **那么** 系统在输入框中插入换行符，不发送消息

### 需求:系统必须支持流式输出

模型回复必须以流式方式逐字显示，提供打字机效果。

#### 场景:流式显示回复
- **当** 模型开始生成回复
- **那么** 系统逐字显示生成的内容，直到完成

#### 场景:流式中断
- **当** 用户在生成过程中点击停止按钮
- **那么** 系统停止接收流式数据，保留已生成的内容

### 需求:系统必须渲染 Markdown 内容

模型回复中的 Markdown 语法必须被正确渲染。

#### 场景:渲染标题和列表
- **当** 模型回复包含 Markdown 标题或列表
- **那么** 系统将其渲染为对应的 HTML 格式

#### 场景:渲染代码块
- **当** 模型回复包含代码块（指定语言）
- **那么** 系统渲染代码块并应用语法高亮

#### 场景:渲染行内代码
- **当** 模型回复包含行内代码（反引号包裹）
- **那么** 系统以等宽字体和背景色显示

### 需求:系统必须显示消息角色

用户消息和助手消息必须有清晰的视觉区分。

#### 场景:区分用户和助手消息
- **当** 对话历史包含用户和助手消息
- **那么** 系统使用不同的样式（位置、颜色、图标）区分两者

### 需求:系统必须支持复制消息内容

用户必须能够复制消息的原始文本或代码。

#### 场景:复制消息
- **当** 用户点击消息的复制按钮
- **那么** 系统将消息内容复制到剪贴板

#### 场景:复制代码块
- **当** 用户点击代码块的复制按钮
- **那么** 系统将代码内容复制到剪贴板
