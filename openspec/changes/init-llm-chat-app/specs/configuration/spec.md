## 新增需求

### 需求:用户可以管理模型提供者配置

系统必须提供界面让用户添加、编辑、删除模型提供者配置。

#### 场景:添加提供者
- **当** 用户点击"添加提供者"并填写配置信息
- **那么** 系统保存配置并使其可被选择使用

#### 场景:编辑提供者
- **当** 用户修改已存在提供者的配置
- **那么** 系统更新配置，后续对话使用新配置

#### 场景:删除提供者
- **当** 用户删除一个提供者配置
- **那么** 系统移除该配置，不再显示在可选列表中

### 需求:系统必须支持配置不同类型的提供者

不同类型的提供者需要不同的配置字段。

#### 场景:配置 llama.cpp 类型
- **当** 用户添加 llama.cpp 类型提供者
- **那么** 系统要求配置端点 URL（如 http://localhost:8080）

#### 场景:配置 OpenAI 类型
- **当** 用户添加 OpenAI 类型提供者
- **那么** 系统要求配置 API Key，可选择配置自定义端点

#### 场景:配置自定义类型
- **当** 用户添加自定义类型提供者
- **那么** 系统要求配置端点 URL 和 API Key

### 需求:用户可以配置模型参数

系统必须允许用户调整模型调用的参数。

#### 场景:设置温度参数
- **当** 用户调整温度（temperature）滑块
- **那么** 后续对话使用新的温度值

#### 场景:设置最大生成长度
- **当** 用户设置最大 token 数
- **那么** 模型生成不超过该长度的回复

#### 场景:设置系统提示词
- **当** 用户编辑系统提示词
- **那么** 后续对话包含该系统提示词

### 需求:系统必须验证配置有效性

系统必须在保存配置前验证其有效性。

#### 场景:验证 API 连接
- **当** 用户保存提供者配置
- **那么** 系统尝试连接 API 验证配置是否正确

#### 场景:显示验证错误
- **当** 配置验证失败
- **那么** 系统显示具体的错误信息帮助用户排查

### 需求:用户可以设置默认提供者和模型

系统必须允许用户设置默认使用的提供者和模型。

#### 场景:设置默认模型
- **当** 用户将某个模型设为默认
- **那么** 新建会话时自动选择该模型

### 需求:系统必须支持全局配置

系统必须提供全局配置功能，存储默认值和系统级设置。

#### 场景:设置全局默认参数
- **当** 用户在设置页面修改默认 temperature 或 max_tokens
- **那么** 新建会话时使用这些默认值

#### 场景:设置全局系统提示词
- **当** 用户配置默认系统提示词
- **那么** 新建会话时自动填充该提示词

#### 场景:会话覆盖全局配置
- **当** 会话设置了特定的 temperature 或系统提示词
- **那么** 使用会话级配置，不使用全局默认值

### 需求:系统必须支持会话级模型参数

每个会话必须能够独立配置模型参数。

#### 场景:会话级温度设置
- **当** 用户在会话设置中修改温度
- **那么** 仅影响当前会话的模型调用

#### 场景:会话级系统提示词
- **当** 用户在会话设置中编辑系统提示词
- **那么** 仅影响当前会话的对话上下文
