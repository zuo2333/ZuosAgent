## 新增需求

### 需求:系统必须支持多种模型提供者

系统必须抽象统一的模型接口，支持本地和云端模型源。

#### 场景:列出可用提供者
- **当** 用户查看模型选择器
- **那么** 系统显示所有已配置且激活的模型提供者

#### 场景:切换提供者
- **当** 用户选择不同的模型提供者
- **那么** 后续对话使用该提供者的模型

### 需求:系统必须支持 llama.cpp 本地模型

系统必须能够连接本地运行的 llama.cpp server。

#### 场景:连接本地 llama.cpp
- **当** llama.cpp server 在配置的端点运行
- **那么** 系统能够调用其 API 进行对话

#### 场景:本地模型不可用提示
- **当** llama.cpp server 未运行或不可达
- **那么** 系统显示友好的错误提示，建议用户检查服务状态

### 需求:系统必须支持 OpenAI API

系统必须能够通过 API Key 连接 OpenAI 服务。

#### 场景:配置 OpenAI 提供者
- **当** 用户添加 OpenAI API Key
- **那么** 系统能够调用 OpenAI 的模型

#### 场景:选择 OpenAI 模型
- **当** 用户选择 OpenAI 提供者
- **那么** 系统显示该账号可用的模型列表（GPT-4, GPT-3.5 等）

### 需求:系统必须支持自定义 API 端点

用户必须能够添加自定义的 OpenAI 兼容 API 端点。

#### 场景:添加自定义提供者
- **当** 用户配置自定义端点 URL 和 API Key
- **那么** 系统将其作为可选的模型提供者

#### 场景:自定义端点兼容性验证
- **当** 用户添加自定义端点
- **那么** 系统验证端点是否支持 OpenAI 兼容 API 格式

### 需求:系统必须统一模型调用接口

所有模型提供者必须通过统一的抽象层调用。

#### 场景:统一消息格式
- **当** 系统调用任意提供者
- **那么** 使用统一的消息格式（role, content）

#### 场景:统一流式响应处理
- **当** 模型返回流式响应
- **那么** 系统将其转换为统一的 SSE 格式发送给前端

### 需求:系统必须加密存储 API Key

敏感信息（如 API Key）必须加密存储，禁止明文存储。

#### 场景:存储 API Key
- **当** 用户保存包含 API Key 的配置
- **那么** 系统使用加密算法存储密钥

#### 场景:读取 API Key
- **当** 系统需要使用 API Key 调用模型
- **那么** 系统解密密钥后使用，不在日志或响应中暴露
