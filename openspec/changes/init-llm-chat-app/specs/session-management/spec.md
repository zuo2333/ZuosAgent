## 新增需求

### 需求:用户可以创建新会话

系统必须允许用户创建新的对话会话。

#### 场景:创建新会话
- **当** 用户点击"新建会话"按钮
- **那么** 系统创建一个新的空会话，并切换到该会话

#### 场景:新会话默认设置
- **当** 创建新会话
- **那么** 会话使用当前选定的模型提供者和模型

### 需求:用户可以查看会话列表

系统必须显示所有会话的列表，便于用户切换。

#### 场景:显示会话列表
- **当** 用户查看侧边栏
- **那么** 系统按更新时间倒序显示所有会话

#### 场景:会话标题自动生成
- **当** 新会话的第一条消息发送后
- **那么** 系统基于首条消息自动生成会话标题（最长 100 字符）

#### 场景:标题长度限制
- **当** 用户编辑会话标题超过 100 字符
- **那么** 系统截断标题并提示长度限制

### 需求:用户可以切换会话

系统必须允许用户在不同会话之间切换。

#### 场景:切换会话
- **当** 用户点击会话列表中的某个会话
- **那么** 系统加载该会话的对话历史并显示

### 需求:用户可以删除会话

系统必须允许用户删除不需要的会话。

#### 场景:删除会话
- **当** 用户点击会话的删除按钮并确认
- **那么** 系统永久删除该会话及其所有消息

#### 场景:删除确认
- **当** 用户请求删除会话
- **那么** 系统显示确认对话框，防止误删

### 需求:系统必须持久化会话数据

会话和消息必须存储在数据库中，刷新页面后仍然可用。

#### 场景:保存消息
- **当** 用户发送消息或收到回复
- **那么** 系统将消息保存到数据库

#### 场景:加载历史会话
- **当** 用户重新打开应用
- **那么** 系统从数据库加载所有会话历史

### 需求:用户可以重命名会话

系统必须允许用户修改会话标题。

#### 场景:重命名会话
- **当** 用户编辑会话标题并保存
- **那么** 系统更新会话标题
