# 长期记忆 (Long-term Memory)

## 新增需求

### 需求:记忆提取

系统必须从会话结束时提取值得长期记忆的信息。

#### 场景:提取事实性记忆
- **当** 用户在对话中陈述"我叫张三"、"我在字节跳动工作"等事实
- **那么** 系统提取为 memory_type="fact" 的记忆

#### 场景:提取偏好性记忆
- **当** 用户表达"我喜欢简洁的回答"、"不要代码注释"等偏好
- **那么** 系统提取为 memory_type="preference" 的记忆

#### 场景:提取事件性记忆
- **当** 对话中发生重要交互（完成项目、解决问题）
- **那么** 系统提取为 memory_type="event" 的记忆

#### 场景:高置信度过滤
- **当** LLM 提取的记忆置信度低于 0.7
- **那么** 系统不存储该记忆

### 需求:向量嵌入

系统必须为每条长期记忆生成向量嵌入。

#### 场景:本地模型嵌入
- **当** 配置使用本地 Embedding 模型
- **那么** 系统使用 bge-large-zh-v1.5 生成 1024 维向量

#### 场景:OpenAI 嵌入
- **当** 配置使用 OpenAI Embedding
- **那么** 系统使用 text-embedding-3-small 生成 1536 维向量

### 需求:语义检索

系统必须支持基于向量相似度的记忆检索。

#### 场景:相似度检索
- **当** 查询"上次项目讨论的内容"
- **那么** 系统返回与查询语义相关的记忆，按相似度排序

#### 场景:类型过滤
- **当** 查询指定 memory_types=["fact", "preference"]
- **那么** 系统只返回指定类型的记忆

#### 场景:Top-K 限制
- **当** 查询指定 top_k=5
- **那么** 系统返回最多 5 条最相关的记忆

### 需求:记忆去重

系统必须避免存储重复或高度相似的记忆。

#### 场景:相似记忆更新
- **当** 新记忆与现有记忆相似度 > 0.85
- **那么** 系统更新现有记忆的重要性而非创建新记忆

### 需求:重要性评分

系统必须为每条记忆维护重要性评分。

#### 场景:初始重要性
- **当** 记忆首次创建
- **那么** 系统根据 LLM 判断设置 importance 值 (0-1)

#### 场景:访问增强
- **当** 记忆被检索并用于上下文
- **那么** 系统增加该记忆的 importance 值

### 需求:遗忘曲线

系统必须实现记忆的时间衰减机制。

#### 场景:时间衰减
- **当** 记忆创建超过 30 天
- **那么** 系统降低其 decay_factor（每 100 天衰减约 63%）

#### 场景:低权重清理
- **当** 记忆的 decay_factor * importance < 0.1
- **那么** 系统标记该记忆为待清理

无

## 移除需求
