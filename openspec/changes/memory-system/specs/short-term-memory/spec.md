# 短期记忆 (Short-term Memory)

## 新增需求

### 需求:会话摘要生成

系统必须在会话消息超过阈值时生成会话摘要，用于压缩上下文。

#### 场景:超过阈值触发压缩
- **当** 会话消息 Token 数超过 3000
- **那么** 系统保留最近 6 条消息，对早期消息生成摘要

#### 场景:摘要增量更新
- **当** 会话已有摘要且再次触发压缩
- **那么** 系统将新消息与现有摘要合并，生成更新后的摘要

### 需求:实体提取

系统必须从对话中提取关键实体（人物、项目、技术等）并存储。

#### 场景:提取人物实体
- **当** 用户在对话中提到"张三"、"李四"等人物名称
- **那么** 系统将人物名称存储到实体的"人物"类别

#### 场景:提取技术实体
- **当** 用户讨论"Python"、"FastAPI"等技术
- **那么** 系统将技术名称存储到实体的"技术"类别

### 需求:任务状态跟踪

系统必须跟踪当前会话的待办任务和意图。

#### 场景:记录待办任务
- **当** 用户说"帮我记住..."或"待会儿要做..."
- **那么** 系统将任务添加到 pending_tasks 列表

#### 场景:任务完成标记
- **当** 用户表示任务已完成
- **那么** 系统更新任务状态为 "completed"

### 需求:Token 统计

系统必须跟踪会话的 Token 使用量，用于压缩决策。

#### 场景:统计消息 Token
- **当** 每条消息添加到会话
- **那么** 系统更新 total_tokens 计数

#### 场景:压缩后更新统计
- **当** 会话完成压缩
- **那么** 系统重新计算 total_tokens（摘要 + 保留消息）
