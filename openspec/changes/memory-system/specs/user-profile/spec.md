# 用户画像 (User Profile)

## 新增需求

### 需求:基础信息管理

系统必须存储和管理用户的基础信息。

#### 场景:获取或创建画像
- **当** 首次访问系统
- **那么** 系统自动创建默认用户画像

#### 场景:更新昵称
- **当** 用户通过 API 设置昵称
- **那么** 系统更新 nickname 字段

#### 场景:设置语言偏好
- **当** 用户设置语言偏好
- **那么** 系统更新 language 字段，默认为 "zh-CN"

### 需求:偏好设置

系统必须存储用户的回复风格偏好。

#### 场景:设置回复风格
- **当** 用户偏好简洁回复
- **那么** 系统存储 response_style.verbosity="concise"

#### 场景:设置技术等级
- **当** 用户技术水平为专家
- **那么** 系统存储 tech_level="expert"

#### 场景:添加兴趣领域
- **当** 用户表达对某领域的兴趣
- **那么** 系统添加到 interests 列表

### 需求:行为特征分析

系统必须自动分析用户的行为特征。

#### 场景:任务类型统计
- **当** 用户完成一次编程相关对话
- **那么** 系统增加 task_distribution["programming"] 的权重

#### 场景:活跃时段分析
- **当** 用户在特定时段活跃
- **那么** 系统更新 active_hours 统计

#### 场景:工具使用统计
- **当** Agent 调用 web_search 工具
- **那么** 系统增加 tool_usage["web_search"] 计数

### 需求:知识图谱

系统必须维护用户相关的实体关系图。

#### 场景:添加实体
- **当** 用户提到"我的同事李四"
- **那么** 系统在知识图谱中添加实体"李四"，类型为"person"

#### 场景:添加关系
- **当** 用户提到"我在字节跳动工作"
- **那么** 系统添加关系：用户 -[就职于]-> 字节跳动

#### 场景:实体属性更新
- **当** 用户补充"李四是后端开发"
- **那么** 系统更新实体"李四"的 attributes

### 需求:画像推断

系统必须从对话中自动推断用户特征。

#### 场景:推断技术水平
- **当** 用户使用专业术语讨论高级主题
- **那么** 系统推断 tech_level 为 "expert"

#### 场景:推断回复偏好
- **当** 用户多次要求"简单点"或"详细点"
- **那么** 系统推断 response_style.verbosity

#### 场景:推断频率控制
- **当** 距离上次推断不足 24 小时
- **那么** 系统跳过本次推断

### 需求:画像版本管理

系统必须支持用户画像的版本管理。

#### 场景:版本递增
- **当** 画像字段更新
- **那么** 系统递增 profile_version

#### 场景:更新时间戳
- **当** 画像任何字段变更
- **那么** 系统更新 updated_at 时间戳

## 修改需求

无

## 移除需求
