## 修改需求

### 需求:背景设计必须简约通透

系统背景必须采用柔和的渐变或纯色设计，禁止使用装饰性动画元素。

#### 场景:默认背景显示
- **当** 用户打开应用
- **那么** 背景显示为柔和的单色渐变，无浮动装饰元素

#### 场景:背景不干扰内容
- **当** 用户浏览聊天内容
- **那么** 背景保持低调，不分散用户注意力

### 需求:侧边栏必须采用浅色通透风格

侧边栏必须使用浅色（白色/淡灰色）玻璃效果，禁止使用深色背景。

#### 场景:侧边栏视觉风格
- **当** 用户查看侧边栏
- **那么** 侧边栏显示为浅色通透效果，与主内容区风格协调

#### 场景:侧边栏可读性
- **当** 用户浏览会话列表
- **那么** 文字和图标清晰可辨，对比度充足

### 需求:圆角必须统一且较大

所有主要组件的圆角必须统一使用 16-28px 范围，禁止使用过小的圆角。

#### 场景:消息气泡圆角
- **当** 用户查看消息
- **那么** 消息气泡显示 20-24px 的圆角

#### 场景:输入框圆角
- **当** 用户聚焦输入框
- **那么** 输入框显示 24px 的圆角

#### 场景:按钮圆角
- **当** 用户悬停按钮
- **那么** 按钮显示 12-16px 的圆角

### 需求:间距必须宽松通透

组件间距必须比当前设计增加至少 20%，禁止使用紧凑布局。

#### 场景:消息列表间距
- **当** 用户滚动消息列表
- **那么** 消息之间有充足的间距，视觉不拥挤

#### 场景:侧边栏会话项间距
- **当** 用户浏览会话列表
- **那么** 会话项之间有适当的间距，易于点击

### 需求:动画必须精简

系统必须仅保留功能性过渡动画，禁止装饰性动画效果。

#### 场景:悬停动画
- **当** 用户悬停按钮或会话项
- **那么** 显示轻微的缩放或颜色过渡动画

#### 场景:无装饰动画
- **当** 用户使用应用
- **那么** 不显示浮动光球、脉冲等装饰性动画
