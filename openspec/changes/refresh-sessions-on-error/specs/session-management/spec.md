## 修改需求

### 需求:会话操作失败时自动同步

系统必须在会话操作失败时自动刷新会话列表，确保前端状态与后端数据库保持同步。

#### 场景:删除不存在的会话时刷新列表

- **当** 用户尝试删除一个不存在的会话
- **那么** 系统必须显示错误提示，并自动刷新会话列表

#### 场景:更新不存在的会话时刷新列表

- **当** 用户尝试更新一个不存在的会话
- **那么** 系统必须显示错误提示，并自动刷新会话列表

#### 场景:点击不存在的会话时清除并刷新

- **当** 用户点击一个不存在的会话
- **那么** 系统必须清除当前会话状态、清空消息列表，并自动刷新会话列表
