## 修改需求

### 需求:消息列表全宽布局
消息列表必须占据主内容区域的全部宽度，通过固定边距控制留白，而不是居中限制。

#### 场景:AI 消息显示
- **当** AI 回复消息
- **那么** 消息从左侧开始显示，与左边距保持 24px 间距

#### 场景:用户消息显示
- **当** 用户发送消息
- **那么** 消息靠右显示，与右边距保持 24px 间距

#### 场景:输入框显示
- **当** 显示输入框
- **那么** 输入框占据全部宽度，左右各保持 24px 边距

#### 场景:Streaming Tool Calls 显示
- **当** 显示 streaming tool calls 区域
- **那么** 该区域占据全部宽度，与消息列表边距一致

### 需求:响应式边距一致性
消息列表和输入框的左右边距必须保持一致。

#### 场景:边距一致性
- **当** 在任意屏幕宽度下
- **那么** 消息列表和输入框的左右边距相同
