## 修改需求

### 需求:工具调用记录持久化

系统必须在保存 assistant 消息时，同时将工具调用记录保存到数据库的 `tool_calls` 字段中。

#### 场景:带工具调用的消息被完整保存

- **当** 大模型响应包含工具调用（如 web_search）
- **那么** 系统必须将工具调用的 id、name、arguments、status、result 保存到消息的 `tool_calls` 字段

#### 场景:历史会话加载时显示工具调用卡片

- **当** 用户重新打开一个包含工具调用的历史会话
- **那么** 系统必须从数据库加载 `tool_calls` 并正确渲染工具调用卡片

#### 场景:无工具调用的消息正常保存

- **当** 大模型响应不包含工具调用
- **那么** 系统正常保存消息，`tool_calls` 字段为 null 或空数组
