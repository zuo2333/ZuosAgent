## 新增需求

### 需求:会话设置弹出面板展示

系统必须将会话设置以弹出面板（popover）形式展示，从右上角设置按钮触发，点击面板外部自动关闭。

#### 场景:点击设置按钮打开面板
- **当** 用户点击会话设置按钮
- **那么** 系统在按钮下方弹出设置面板

#### 场景:点击外部关闭面板
- **当** 设置面板已打开
- **且** 用户点击面板外部的任意位置
- **那么** 系统关闭设置面板

#### 场景:再次点击按钮切换面板
- **当** 设置面板已打开
- **且** 用户再次点击设置按钮
- **那么** 系统关闭设置面板

### 需求:弹出面板布局规范

系统必须按照指定布局渲染弹出面板：宽度 ~400px，固定高度 ~500px，内容超出时内部滚动。

#### 场景:面板尺寸正确
- **当** 设置面板打开
- **那么** 面板宽度为 400px（小屏幕上为 calc(100vw - 32px)）
- **且** 面板最大高度为 500px（不超过 calc(100vh - 80px)）
- **且** 内容超出时面板内部滚动

#### 场景:面板定位正确
- **当** 设置面板打开
- **那么** 面板相对于触发按钮定位
- **且** 面板右上角与按钮右对齐

### 需求:单页分区布局

系统必须以单页形式展示所有设置项，使用分隔线区分不同区域：模型选择、参数调整、工具设置。

#### 场景:区域分隔清晰
- **当** 设置面板打开
- **那么** 模型选择区域在顶部
- **且** 参数调整区域在中间
- **且** 工具设置区域在底部
- **且** 各区域间有分隔线

### 需求:视觉风格统一

系统必须使弹出面板的视觉风格与总设置页面保持一致。

#### 场景:样式一致性
- **当** 设置面板打开
- **那么** 面板使用与 SettingsPage 相同的圆角（16px）
- **且** 使用相同的阴影样式
- **且** 使用相同的 CSS 变量（背景、边框、文字颜色）

#### 场景:标题栏样式一致
- **当** 设置面板打开
- **那么** 标题栏使用与 SettingsPage Section Header 相同的样式
- **且** 包含标题文字和关闭按钮

### 需求:工具设置样式适配

系统必须使 ToolSettings 组件使用 CSS 变量而非硬编码颜色，以支持主题一致性。

#### 场景:工具设置使用 CSS 变量
- **当** 设置面板打开
- **那么** ToolSettings 组件使用 CSS 变量定义颜色
- **且** 文字颜色使用 var(--token-text-primary)
- **且** 边框颜色使用 var(--token-border-light)
- **且** 背景颜色使用 var(--token-main-surface-primary)

### 需求:未保存更改提示

当用户有未保存的更改时，系统必须在关闭面板前提示用户。

#### 场景:有未保存更改时关闭面板
- **当** 用户修改了设置项
- **且** 未点击保存
- **且** 用户尝试关闭面板
- **那么** 系统提示用户有未保存的更改
