## 为什么

当前前端显示工具调用时，如果一个 assistant 消息包含多个 tool_calls，它们会合并显示在同一条消息中。这导致：

1. **视觉不一致**：旧会话（每轮一个工具调用）和新会话（一轮多个工具调用）显示效果不同
2. **用户体验差**：多个工具调用堆在一起，不易区分

**期望效果**：每个工具调用作为独立的消息块显示，无论后端如何存储。

## 变更内容

- **修改前端渲染逻辑**：将 assistant 消息中的多个 tool_calls 拆分为独立的消息块
- **保持后端不变**：后端存储逻辑符合 OpenAI API 规范，无需修改

- `message-display`: 修改消息渲染逻辑，每个工具调用独立显示
