## 修改需求

### 需求:工具调用在消息流中展示

系统必须在消息流中展示工具调用过程，而不是在页面底部的临时区域。工具调用卡片必须从"运行中"状态到"完成"状态在原地转变，禁止位置变化导致的视觉跳动。

#### 场景:工具调用开始时在消息流展示

- **当** AI 开始调用工具（如 Web Search）
- **那么** 系统在消息列表末尾展示工具调用卡片，状态显示为"Running"

#### 场景:工具调用状态原地转变

- **当** 工具调用从"运行中"变为"成功"或"失败"
- **那么** 工具卡片在原位置更新状态徽章，显示"SUCCESS"或"FAILED"
- **那么** 工具卡片不发生位置移动

#### 场景:工具调用完成后AI回复追加显示

- **当** 工具调用完成且 AI 开始输出回复内容
- **那么** 流式消息内容在工具卡片下方展示
- **那么** 工具卡片保持原位置不变

#### 场景:多个工具调用顺序展示

- **当** AI 连续调用多个工具
- **那么** 每个工具调用卡片依次在消息流末尾追加展示
- **那么** 已完成的工具卡片保持原位置，新工具卡片在其下方展示
