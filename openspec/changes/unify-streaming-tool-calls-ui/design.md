## 上下文

当前有两个独立的工具调用展示组件：
- `StreamingToolCalls.tsx`：用于显示运行中的工具调用，使用彩色背景 + Emoji 风格
- `ToolCallCard.tsx`：用于显示已完成的工具调用，使用白色卡片 + SVG 图标风格

两者视觉风格不统一，用户在工具执行过程中和完成后看到截然不同的 UI。

## 目标 / 非目标

**目标：**
- 统一 StreamingToolCalls 与 ToolCallCard 的视觉风格
- 多个工具调用堆叠显示，节省空间
- 用户能清晰识别有多个工具正在运行
- 运行中的工具展开显示详情，完成的工具折叠

**非目标：**
- 不改变后端工具调用逻辑
- 不实现真实的进度条（后端不提供进度信息）
- 不改变 ToolCallCard 的现有设计

## 决策

### 决策 1：复用 ToolCallCard 样式常量

**选择**：从 ToolCallCard.tsx 提取共享常量（TOOL_ICONS、TOOL_NAMES、STATUS_CONFIG），在两个组件间复用。

**理由**：避免重复代码，确保样式一致性。

**替代方案**：
- 复制样式到 StreamingToolCalls → 会导致维护困难，样式容易漂移
- 创建共享样式文件 → 增加文件数量，当前规模不需要

### 决策 2：移除假进度条

**选择**：移除固定 60% 宽度的假进度条，改用耗时计时 + 旋转动画表示运行状态。

**理由**：
- 假进度条误导用户，让人以为有真实进度
- 计时器提供有用信息（已执行多久）
- ChatGPT 风格也是用旋转动画，无进度条

### 决策 3：堆叠策略

**选择**：每个工具调用独立卡片，垂直堆叠。运行中的默认展开，已完成的默认折叠。

**理由**：
- 用户能看到每个工具的状态
- 展开状态让用户知道"正在发生什么"
- 折叠已完成工具节省空间

**替代方案**：
- 分组指示器（3 tools running）→ 需要额外点击查看详情，信息密度低

### 决策 4：运行中状态徽章样式

**选择**：运行中状态使用蓝色圆点 + "Running" 文字，与 success 状态徽章风格一致。

**样式定义**：
- 背景色：`#eff6ff`（浅蓝）
- 文字颜色：`#2563eb`（蓝色）
- 圆点颜色：`#3b82f6`（亮蓝）
- 圆点动画：旋转动画（与 ToolCallCard running 状态一致）

### 决策 5：状态转换动画

**选择**：状态变化时添加平滑过渡动画。

**动画细节**：
- 圆点颜色渐变（蓝 → 绿/红）
- 徽章背景色渐变
- 文字颜色渐变
- 过渡时长：200ms
- 缓动函数：`cubic-bezier(0.4, 0, 0.2, 1)`（与项目 CSS 变量一致）

**理由**：动画提供视觉反馈，让用户感知状态变化，同时保持 ChatGPT 风格的简洁感。

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| 运行中工具数量多时可能占用大量空间 | 设置最大高度 + 滚动 |
| SVG 图标可能比 Emoji 加载稍慢 | 图标很小，影响可忽略 |
