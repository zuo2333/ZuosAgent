## 修改需求

### 需求:工具调用卡片视觉一致性
运行中的工具调用卡片必须与已完成的工具调用卡片（ToolCallCard）使用相同的视觉风格。

#### 场景:卡片背景和边框一致
- **当** 工具调用处于运行中或已完成状态
- **那么** 卡片必须使用白色背景（#ffffff）和浅灰边框（1px solid #e6e6e6）

#### 场景:状态徽章样式一致
- **当** 显示工具调用状态
- **那么** 必须使用小圆点 + 文字标签的徽章样式，与 ToolCallCard 一致

### 需求:参数行内展示
工具调用的参数必须以行内 `key="value"` 格式展示，而非 JSON 字符串格式。

#### 场景:参数格式化显示
- **当** 工具调用包含参数
- **那么** 参数必须以 `key="value", key2="value2"` 的行内格式展示

### 需求:运行状态指示
运行中的工具调用必须显示耗时计时器，而非假进度条。

#### 场景:显示耗时
- **当** 工具调用处于运行中状态
- **那么** 必须显示从开始执行到当前的耗时（秒）

#### 场景:禁止假进度条
- **当** 后端未提供真实进度信息
- **那么** 禁止显示固定宽度的假进度条

### 需求:多工具堆叠显示
当存在多个工具调用时，必须垂直堆叠显示，每个工具独立卡片。

#### 场景:多个工具独立显示
- **当** 同时有多个工具调用
- **那么** 每个工具必须显示为独立的卡片，垂直堆叠

#### 场景:运行中展开已完成折叠
- **当** 存在运行中和已完成的工具调用
- **那么** 运行中的工具默认展开显示详情，已完成的工具默认折叠

### 需求:SVG图标替代Emoji
工具图标必须使用 SVG 图标，而非 Emoji。

#### 场景:Web Search 图标
- **当** 工具类型为 web_search
- **那么** 必须使用放大镜 SVG 图标

#### 场景:Database Query 图标
- **当** 工具类型为 db_query
- **那么** 必须使用数据库 SVG 图标

### 需求:运行中状态徽章样式
运行中状态徽章必须与成功状态徽章风格一致，使用蓝色调。

#### 场景:运行中徽章样式
- **当** 工具调用处于运行中状态
- **那么** 徽章必须显示蓝色圆点 + "Running" 文字，背景为浅蓝色（#eff6ff）

#### 场景:运行中圆点动画
- **当** 工具调用处于运行中状态
- **那么** 圆点必须显示旋转动画

### 需求:状态转换动画
状态变化时必须提供平滑的过渡动画。

#### 场景:徽章颜色渐变
- **当** 工具状态从运行中变为成功或失败
- **那么** 徽章的背景色、文字颜色、圆点颜色必须在 200ms 内渐变过渡
